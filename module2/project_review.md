# ⚙️ 효율적인 인사 및 학습 관리를 위한 LMS 마이페이지 시스템 구축기 (Module 2)
> **핵심 요약:** JPA 무결성 유지, 복합 테이블 연쇄 삭제 로직, Full Calendar API 기반의 근태 및 연차 관리 시스템 구현

---

## 1. 개요
모듈 2에서는 사용자의 개인화 경험을 극대화하는 **마이페이지** 전반과, 인사 관리의 핵심인 **출결/연차 시스템**을 담당했습니다. 특히 관리자와 사용자 간의 데이터 동기화와 JPA를 활용한 복잡한 데이터 처리 로직을 안정화하는 데 집중했습니다.

---

## 2. 해결 과정 및 설계 의도 (Solution & Design)

### 2-1. 스마트 근태 및 연차 관리 (Calendar 기반)
* **Full Calendar 기반 UI/UX**: JS를 활용해 실제 동작하는 달력을 구현하고, 출근/지각/결근/공결 상태를 형광펜 형식으로 시각화했습니다.
* **지능형 연차 로직**: 사용자가 선택한 날짜 범위를 계산하여 보유 연차를 자동 차감하며, 오늘 이전 날짜 신청 방지 및 잔여 연차 초과 시 차단하는 검증 로직을 구현했습니다.
* **자동 출결 처리**: 별도의 체크 버튼 없이 서비스 진입 시 로그인 기록을 바탕으로 당일 출결 데이터를 생성하여 사용자 편의성을 높였습니다.

### 2-2. 관리자 전용 출결 수정 시스템
* **대상자 타겟팅**: `{userId}` 경로 변수를 활용해 특정 사용자의 출결 데이터를 캘린더에 로드하고 수정할 수 있는 관리자 기능을 구현했습니다.
* **드롭다운 수정 인터페이스**: 캘린더 내 수정할 날짜를 선택하면 **드롭다운 메뉴**가 활성화되어 출결 상태를 직관적으로 변경할 수 있게 설계했습니다.
* **데이터 무결성 제약**: 출결 테이블에 기록이 없는 날짜는 수정을 제한하여 데이터 정합성을 확보했습니다.

### 2-3. 포인트 및 멤버십 시스템
* **동적 포인트 바**: 문제 풀이 및 강의 수강 결과에 따라 실시간으로 변동되는 포인트 바(Progress Bar)를 구현했습니다.
* **직급 가이드**: 포인트별 직급 정보와 다음 단계까지 남은 포인트 산출 로직을 통해 사용자에게 학습 동기를 부여했습니다.

### 2-4. 안전한 회원 탈퇴 로직
* **연쇄 삭제(Cascade) 처리**: 자식 테이블(댓글, 제출 문제, 게시글, 강의 섹션 등)의 데이터를 순차적으로 삭제하여 DB 무결성 제약 조건 위반을 방지했습니다.

---

## 3. 구조 및 데이터 흐름 (Architecture & Data Flow)

### 3-1. 관리자 출결 수정 Flow
관리자가 특정 사용자의 출결 상태를 변경하고 DB에 반영하기까지의 데이터 흐름입니다.

1. **Request**: 브라우저 주소창의 `{userId}`와 선택한 날짜 데이터 전달
2. **Identification**: `principal`을 통해 관리자 권한 확인 및 대상 유저 정보 로드
3. **Validation**: 해당 날짜에 기존 출결 데이터가 존재하는지 확인 (기록 없는 날짜 수정 불가)
4. **Processing**: JPA의 PK(기본키)를 포함한 엔티티 구성을 통해 신규 생성이 아닌 `Update` 수행
5. **Response**: DB 반영 후 업데이트된 출결 현황 리턴

## 4. 트러블슈팅 (Troubleshooting)

> **❌ JPA 데이터 업데이트(Update) 대신 삽입(Insert)되는 이슈**
> * **현상**: 관리자가 출결 수정 시 기존 데이터가 바뀌지 않고 계속 새로운 데이터가 쌓임.
> * **원인**: 엔티티 객체에 PK(기본키) 정보가 누락된 채로 `save()`가 호출되어 JPA가 이를 새로운 데이터로 인식함.
> * **해결**: `UsersDTO` 생성자에 `userId`를 추가하고, `attendanceId` 등 PK를 포함한 삼중 구조(ID, User, Date) 비교 로직을 적용하여 정확한 `Update`를 보장했습니다.

> **❌ DB 예약어 충돌로 인한 쿼리 오류**
> * **현상**: 연차 사용 기능 테스트 중 SQL 문법 오류 발생.
> * **원인**: 엔티티의 필드명인 `rank`가 MySQL의 예약어(Function)와 동일하여 충돌 발생.
> * **해결**: `@Column(name = "\`rank\`")`와 같이 백틱을 사용하여 필드명을 감싸 예약어 충돌 문제를 해결했습니다.

> **❌ 회원 탈퇴 시 참조 무결성 오류**
> * **현상**: 특정 테이블(`sectionprogress`)에 `userId` 컬럼이 없어 전체 삭제 로직이 중단됨.
> * **원인**: 직접적인 연관 관계가 없는 테이블에서 상위 엔티티 삭제를 시도함.
> * **해결**: 해당 테이블이 참조하는 `enrollmentId`를 먼저 조회한 후, `deleteByEnrollmentId_UserId`와 같이 명확한 명명 규칙을 활용한 삭제 메서드로 로직을 개선했습니다.

## 5. 결과 및 성과 (Results)
* **안정성**: JPA의 동작 원리를 정확히 파악하여 데이터 중복 생성 문제를 해결하고 무결성 확보.
* **UX 개선**: 드롭다운 UI와 실시간 포인트 바를 통해 관리자와 일반 사용자 모두의 편의성 증대.
* **환경 관리**: `taskkill` 및 `SHOW PROCESSLIST` 등을 활용하여 서버와 DB 세션 꼬임 문제를 스스로 해결하며 운영 환경 이해도 향상.

## 6. 리팩토링 (Refactoring)

### 📋 출결 데이터 생성 로직 최적화
* **AS-IS**: 출결 조회 시 시간 정보까지 비교하여 동일 날짜에 중복 데이터가 생성되는 문제 발생.
* **TO-BE**: 시간 정보를 제외한 **날짜(Date) 기반 비교 로직**으로 개선하여 하루에 단 하나의 출결 데이터만 유지되도록 강화했습니다.
