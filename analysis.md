# 🎯 백엔드 개발자 직무 분석
 
> 국내 주요 기업들의 백엔드 채용 공고 및 직무 설명을 분석해, 백엔드 개발자에게 공통적으로 요구되는 핵심 자질을 정리했습니다.
 
---
 
## 백엔드 개발자
 
| 항목 | 내용 |
|---|---|
| **Objective** | 서비스의 핵심 데이터와 트랜잭션을 안정적으로 처리하고, 트래픽 증가에도 견고하게 확장 가능한 시스템을 구축 |
| **해결 문제** | 트래픽 폭증 시에도 무너지지 않는 구조 설계, 데이터 정합성 유지, 서비스 중단 없는 배포·운영 |
| **성과지표(KR)** | 서버 응답 속도, 장애 복구 시간, 트래픽 처리량(TPS), 시스템 가용성(Uptime), 배포 시간 단축 |
| **고객/시장** | B2C·B2B를 아우르는 서비스 사용자 / 산업군에 관계없이 데이터 기반 서비스가 존재하는 모든 시장 |
| **조직 단계** | 스타트업의 초기 구조 설계부터, 성숙 기업의 고도화·글로벌 확장 단계까지 전 단계에서 요구됨 |
| **핵심 역량** | API/DB 설계 및 최적화, 성능 개선(캐싱·쿼리 튜닝), 시스템 안정성에 대한 집착, CS 기본기, CI/CD·인프라 자동화, 분산 시스템(MSA) 이해 |
| **기술 스택** | `Java`/`Kotlin`/`Go`, `Spring Boot`, `MySQL`, `Redis`, `Docker`, `Kubernetes`, `AWS`/`Azure`, `Kafka` |
 
---
 
## 📌 종합
 
위 프로필에서 드러나는 공통점은, 백엔드 개발자에게 요구되는 역량이 **"기능을 구현하는 능력"에서 "서비스를 안정적으로 유지·확장하는 능력"으로 이동**하고 있다는 점입니다. 성능 최적화와 인프라 자동화는 과거엔 DBA·DevOps라는 별도 직무의 영역으로 여겨졌지만, 최근에는 백엔드 개발자 본연의 역량으로 통합되는 흐름을 보이고 있으며, 이는 채용 공고의 "핵심 역량" 항목에서도 일관되게 확인됩니다.
---

## 🚩 향후 실행 계획
> 11개 기업 공고를 바탕으로 분석한 백엔드 개발자의 핵심 역량을 확보하기 위한 단계별 학습 로드맵입니다.

### 1️⃣ 알고리즘 및 CS 기초 (현재 진행 중)
* **Coding Test**: `Python`을 주력 언어로 선정, **백준(Baekjoon)** 플랫폼을 통해 단계별 문제 풀이 및 실전 감각 익히기
* **CS Knowledge**: **정보처리기사 실기** 준비를 병행하며 `SQL`, `C언어`, `Java` 등 다중 언어의 기초 문법과 데이터베이스 구조 학습

### 2️⃣ 기술적 복기 및 심화 (Short-term)
* **Deep Learning (PyTorch)**: 학부 논문 당시 수행했던 **`VGG16` 모델 파이프라인** 정밀 복기
  * 단순 모델 활용을 넘어, `PyTorch` 환경에서의 데이터 전처리 및 모델 계층(Layer) 구조 재분석
  * 학습 과정에서 발생했던 이슈와 해결 과정을 기술적으로 문서화하여 면접 대비

### 3️⃣ 데이터 전문성 강화 (Long-term)
* **Certificate**: **빅데이터분석기사** 취득을 목표로 설정
  * 대용량 데이터 처리 프로세스와 통계적 분석 역량을 검증하고 실무 데이터 다루는 법 내재화
* **Focus**: 분석 리포트에서 도출된 기업들의 '데이터 기반 의사결정' 역량을 갖추기 위해 지표 설계 및 분석 방법론 지속 학습
