# 📔 모듈 3: 웹 고도화 및 아키텍처 실무 기록
> 웹 애플리케이션의 안정성을 높이기 위한 예외 처리 및 로깅 아키텍처부터, 가벼운 웹 계층 테스트 환경 구축, 그리고 도메인 주도 설계(DDD) 및 이벤트 스토밍을 통한 현대적 마이크로서비스 기초 설계까지 실무 기술을 학습하고 적용한 기록입니다.

---

## 🌐 RESTful API 심화 및 유효성 검증 (Validation)

### 1. @RestController와 응답 제어 표준화
* **단일 책임 원칙(SRP) 기반 계층 분리**: `@Controller`는 View를 반환하는 책임이 있는 반면, 데이터를 전송하는 웹 계층은 데이터 응답에만 집중할 수 있도록 `@RestController`를 사용하여 역할을 명확히 분리합니다[cite: 1].
* **Jackson 라이브러리와 자동 파싱**: Spring 부트에 내장된 Jackson 라이브러리가 Java 객체와 JSON 포맷 간의 상호 파싱을 자동으로 처리하므로, 객체를 그대로 반환하여 웹 통신 표준을 준수합니다.
* **ResponseEntity를 통한 정밀 제어**: HTTP 응답 헤더, 상태 코드, 바디를 유연하게 제어하기 위해 `ResponseEntity<?>` 와일드카드를 사용하며, 리소스 반환 여부에 관계없이 생성자 대신 안정적인 빌더(Builder) 패턴 활용을 지향합니다.
* **HATEOAS 기반 3단계 RESTful API**: API 호출 시 관련 리소스 링크 정보(`linkTo`, `methodOn`, `withSelfRel`, `withRel`)를 충분히 제공함으로써 클라이언트가 자율적으로 애플리케이션 상태를 탐색할 수 있는 진정한 RESTful 아키텍처를 구현합니다.

### 2. 예외 처리 아키텍처 및 입력값 방어
* **전역 예외 처리(Global Exception Handling)**: 비즈니스 로직 및 유효성 검증 실패 시 발생하는 예외를 `ExceptionControllerAdvice` 및 `@ControllerAdvice` 내부에서 포착하고, 일관된 오류 포맷(`ErrorResponse`)으로 정형화하여 시스템 복원력을 높입니다.
* **Bean Validation 2차 방어**: 클라이언트 단의 침투 및 변조 가능성을 차단하기 위해 백엔드 서버에서 `@Valid` 어노테이션을 통해 데이터 유효성을 엄격하게 검증합니다.
    * **필수 데이터 및 공백 제어**: `@NotNull` (null 불허), `@NotEmpty` (null, "" 불허), `@NotBlank` (null, "", " " 모두 불허).
    * **형식 및 범위 제한**: `@Email` (이메일 검증), `@Size` (길이 제한), `@Min`/`@Max` (수치 제한), `@Future`/`@Past` (날짜 검증).
* **커스텀 검증기(Custom Validator)**: 표준 어노테이션으로 해결할 수 없는 특수한 요구사항(예: 평점 단위가 1~5 사이에서 0.5 단위로만 입력되는지 여부 등) 처리를 위해 커스텀 어노테이션(`@ValidRating`)과 연계된 전용 `Validator` 구현체를 개발 및 배치합니다.

## 🪵 애플리케이션 안정성 확보 및 로깅 시스템

### 1. 실무 로깅(Logging) 정책 및 구조
* **Logback과 SLF4J 통합**: Spring Boot 표준인 Logback 프레임워크를 기반으로 하며, 개발 환경(`debug`, `info`, `warn`, `error`)과 운영 단계(`info`, `warn`, `error`)에 최적화된 로그 레벨 정책을 체계화합니다.
* **로그 레벨 분리 기준**:
    * `ERROR`: 즉각적인 시스템 대응이 필요한 장애 상황.
    * `WARN`: 잠재적인 시스템 상의 문제나 비정상적 흐름.
    * `INFO`: 중요 비즈니스 이벤트 기록 (운영 표준).
    * `DEBUG`/`TRACE`: 개발 과정의 상세 흐름 분석 및 디버깅용 정보.

### 2. MDC 기반 분산 요청 추적
* **MDC(Mapped Diagnostic Context)**: 대규모 동시 요청 환경에서 다수의 클라이언트가 인입되더라도 각각의 요청을 고유하게 식별할 수 있는 분산 추적 기법을 적용합니다.
* **OncePerRequestFilter와 TraceId 구현**: HTTP 요청당 단 한 번만 실행되는 `OncePerRequestFilter`를 빈으로 등록하여, 필터 체인 내부의 중복 실행과 컨텍스트 전파 꼬임을 원천 방지하고 요청 고유의 식별자(`traceId`)를 발급 및 주입하여 추적 무결성을 보장합니다.

## 🧪 테스트 주도 개발(TDD) 및 슬라이스 테스트

### 1. 웹 계층 슬라이스 테스트 환경 구축
* **@WebMvcTest 기반 경량화**: 전체 애플리케이션 컨텍스트를 로드하지 않고 컨트롤러를 포함한 가벼운 웹 계층만 독립적으로 로드하여 가볍고 신속한 검증 환경을 구축합니다.
* **MockMvc 가상 요청 시뮬레이션**: 실제 Tomcat WAS 서버를 가동하지 않는 환경에서도 HTTP 요청과 응답을 정밀하게 모의 실행(`MockMvc`)하여 엔드포인트의 입력과 출력(JSON 파싱, 상태 코드 검증) 구조를 신속하게 테스팅합니다.
* **@MockitoBean 가짜 객체 제어**: 실제 의존하는 Service 컴포넌트 대신 Mock(가짜) 객체를 생성하고 계층 간 결합을 격리하여, 컨트롤러의 자체 비즈니스 흐름과 예외 처리 분기만을 정확하게 테스트합니다.

### 2. 프로파일(Profile) 기반 전략 및 검증 구조
* **@ActiveProfiles 환경 격리**: 통합 테스트 진행 시 운영(`prod`)이나 개발(`dev`) 설정과 격리된 독립적인 테스트용 설정 프로파일(`@ActiveProfiles("test")`)을 적용함으로써 외부 DB나 인프라에 영향을 주지 않는 격리 구조를 확보합니다.
* **Given-When-Then 패턴 적용**: 테스트 코드를 준비(Given), 실행(When), 검증(Then)의 명확한 흐름으로 구조화하여 정상계 시나리오뿐만 아니라 시스템 경계값이나 비정상 예외 상황 처리에 대한 안정성을 보증합니다.

## 🏗️ 도메인 주도 설계(DDD) 및 마이크로서비스 기초

### 1. 이벤트 스토밍(Event Storming) 기반 전략적 설계
* **도메인 경계 분리 워크숍**: 도메인 전문가, 고객, 개발자 등 이해관계자가 참여하는 협업 브레인스토밍을 통해 도메인 지식을 시각화하고 바운디드 컨텍스트(Bounded Context) 간의 경계를 유기적으로 추출합니다.
* **유비쿼터스 언어(Ubiquitous Language)**: 도메인 전반에서 통용되는 공통 용어 사전과 도메인 언어를 완벽히 일치시켜 의사소통 비효율성과 비즈니스 요구사항 파악 오류를 최소화합니다.
* **포스트잇 기반 타임라인 정렬 흐름**:
    * **도메인 이벤트 (주황색)**: 비즈니스 프로세스 내에서 발생한 중요한 변경 및 의미 있는 사실을 뜻하며, 반드시 명제적 **과거형** 문장으로 정의합니다.
    * **커맨드 (파란색)**: 사용자가 시스템의 UI 등을 통해 실행하는 명령이나 상호작용적 행위를 의미하며, 주체가 명확한 **현재형** 동사 형태로 설계합니다.

### 2. 상태 비저장(Stateless) 아키텍처와 JWT 인증 구조
* **세션(Stateful)의 확장성 제어**: 단일 서버 환경과 달리 다중 백엔드 서버를 수평 확장(Scale-out)할 때 발생하는 세션 동기화 및 메모리 부하 문제를 근본적으로 타파하기 위해 토큰 기반 인증 아키텍처로 변환합니다.
* **JWT(JSON Web Token) 명세와 인증 매커니즘**:
    * **Header**: 토큰 타입(JWT)과 서명에 사용된 해싱 알고리즘 기재.
    * **Payload**: 클라이언트 및 서비스 간 교환할 실제 사용자 식별 정보와 권한 데이터(`Claim`)를 자바스크립트 객체 표현식으로 압축 저장.
    * **Signature**: 암호화 비밀키(Secret Key)와 인코딩 값을 조합하여 무결성 변조 여부를 검증하는 강력한 단방향 보안 메커니즘을 적용.
* **MSA 환경으로의 확장**: 하나의 바운디드 컨텍스트가 곧 독립적인 하나의 애플리케이션 서버로 매핑되는 마이크로서비스 환경에서, API 게이트웨이를 통한 통합 토큰 인증/인가 체계 및 서비스 간 이벤트 소통 구조의 기술적 기틀을 다졌습니다.
